Visão Geral

Este processo de desenvolvimento foi projetado para uma equipe enxuta, com disponibilidade de horários variáveis, priorizando comunicação assíncrona, organização do fluxo de trabalho e redução de retrabalho.

O modelo combina práticas de Kanban para gestão contínua, Design First para validação técnica antecipada e uma esteira de CI/CD que automatiza testes e implantação.

Descrição das Atividades

A. Planejamento e Alinhamento

  • Reunião semanal única
  • Review das entregas anteriores
  • Planejamento e priorização do backlog
  • Discussão de impedimentos técnicos

B. Análise e Design

  • Documentação técnica antes do código
  • Definição de contratos de API e modelagem de dados
  • Redução de retrabalho

C. Desenvolvimento e Testes Locais

  • Uso de feature branches
  • Implementação acompanhada de testes unitários

D. Revisão de Código

  • Pull Request assíncrono
  • Revisão por outro membro da equipe
  • Validação do código e da documentação

E. Pipeline Automatizado (CI/CD)

  • Execução automática de testes
  • Verificação de conflitos
  • Deploy em ambiente de homologação

Papéis e Responsabilidades

  • Product Owner / Líder: gerencia o backlog, define prioridades e valida entregas.
  • Desenvolvedor Full Stack: atua em análise, desenvolvimento, testes e code review.
  • Engenheiro DevOps: mantém infraestrutura, CI/CD e monitora o sistema.

Práticas Ágeis Aplicadas

  • Kanban: fluxo contínuo e limitação de WIP.
  • Design Up-Front Ágil: documentação técnica obrigatória.
  • CI/CD: automação total de testes e deploy.
  • Peer Review: qualidade e compartilhamento de conhecimento.
  • Feature Branch Workflow: trabalho paralelo seguro.
